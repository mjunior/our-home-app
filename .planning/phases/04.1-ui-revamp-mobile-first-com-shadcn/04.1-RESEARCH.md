# Phase 4.1: UI Revamp Mobile-First com shadcn - Research

**Researched:** 2026-03-03
**Domain:** Mobile-first UX polishing with shadcn/ui + Tailwind for financial dashboard journey
**Confidence:** HIGH

## User Constraints

Constraints locked from `04.1-CONTEXT.md`:
- Revamp profundo nesta fase deve focar na jornada `Fluxo de Caixa`.
- Demais telas recebem apenas consistência leve (sem revamp total agora).
- Mobile deve usar CTA principal fixo inferior (sticky bottom action).
- Densidade visual equilibrada (nem compacta demais, nem espaçada demais).
- Motion deve ter sensação premium/viral, sem perder legibilidade.
- Direção visual mantém clean, moderna, memorável, com dark mode obrigatório.
- Navegação lateral mobile-first deve ser mantida.
- Base shadcn/ui + Tailwind já existente deve ser aprofundada, não substituída.

Deferred/out-of-scope:
- Revamp profundo de contas/cartões/categorias/parcelas em fase posterior.
- Mudanças de persistência ou lógica de domínio fora de UX/UI.

## Summary

Phase 4.1 should avoid broad redesign across the whole app and instead deliver a premium, mobile-first experience on the most frequent decision flow (`Fluxo de Caixa`). The winning strategy is to enforce a stable visual grammar: sticky action zone, clear card hierarchy, stronger state transitions, and a consistent token-driven system.

Given the current architecture, the safest approach is incremental: keep existing business logic and tests intact, refine interaction and visual composition through shadcn primitives and dedicated presentation wrappers. This minimizes regression risk while improving perceived quality significantly.

**Primary recommendation:** Ship the revamp in three slices — shell/tokens hardening, deep cashflow journey redesign, and motion + UX validation with focused e2e assertions.

## Architecture Patterns

### Pattern 1: AppShell-first navigation contract
- Keep route state and theme toggles centralized in shell.
- Mobile drawer + quick tab affordance for high-frequency routes.
- Avoid page-level duplication of navigation behaviors.

### Pattern 2: Journey-centric composition
- Treat cashflow page as layered blocks:
  - KPI strip (saldo livre + risk)
  - alert/action block
  - breakdown/cause explainability block
  - operational forms and statements
- Sticky CTA remains visible independent of scroll depth.

### Pattern 3: Motion as hierarchy enhancer
- Use subtle staggered reveals for sections.
- Animate state transitions (risk badge, alert severity) rather than full-page flashy motion.
- Respect reduced-motion preference by disabling non-essential transforms.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Drawer behavior | Custom overlay/open-close logic | Radix Sheet wrapper already in `ui/sheet` | Accessibility + reliability |
| Variant-heavy buttons/badges | Raw class strings scattered in pages | `cva` variants in `ui/button`, `ui/badge` | Keeps style system maintainable |
| Sticky CTA behavior | Per-page ad-hoc fixed footer | Shared `StickyActionBar`-style component in foundation layer | Consistent thumb-zone behavior |

## Common Pitfalls

### Pitfall 1: Over-animating the cashflow journey
- **What goes wrong:** visual noise harms readability and perceived speed.
- **Avoid:** animate only hierarchy/signals (entry, severity, action visibility).

### Pitfall 2: Mobile CTA colliding with forms/lists
- **What goes wrong:** bottom button blocks inputs or table rows.
- **Avoid:** add safe-area padding and bottom spacing in page content.

### Pitfall 3: Inconsistent dark-mode contrast on financial metrics
- **What goes wrong:** key numbers lose emphasis or fail accessibility.
- **Avoid:** enforce tokenized foreground/background pairs for KPI and alert states.

## Code Examples

### Sticky action baseline
```tsx
<div className="fixed inset-x-0 bottom-0 z-30 border-t bg-white/95 p-3 pb-[calc(env(safe-area-inset-bottom)+0.75rem)] dark:bg-slate-950/95">
  <Button className="h-12 w-full">Nova transacao</Button>
</div>
```

### Reduced motion guard
```css
@media (prefers-reduced-motion: reduce) {
  .motion-premium {
    animation: none !important;
    transition: none !important;
  }
}
```
